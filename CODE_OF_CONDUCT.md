# Code of Conduct

## Our Standards

Examples of behavior that contributes to a positive environment:

- Using the project for its intended purpose: building an AI companion
- Giving feedback that is constructive and actionable
- Reporting security vulnerabilities through proper channels
- Keeping discussions relevant to the project's goals

Examples of behavior that will not be tolerated:

- Issues that just say "doesn't work" with no details
- Asking for features that contradict the project's architecture
- Demanding free labor because "it's just AI"
- Treating the maintainer like a 24/7 support hotline

## Purpose

This project is maintained by a single developer in their spare time. They have:

- A full-time job
- A life outside code
- Limited bandwidth for hand-holding

If you submit an issue, include reproduction steps. If you want a feature, explain why it belongs in *this* project rather than your own fork.

## Enforcement

If someone is being unreasonable, the maintainer reserves the right to close the issue, ignore the PR, or block the user. This isn't personal — it's just time management.

## Contact

For security vulnerabilities, reach out privately. For everything else, open an issue and be respectful.
